Current Data Flow (as of now)

API workspace: C:/Users/Server/Documents/API

1) Local PC (source update)

Windows Task Scheduler runs tools/refresh_seed.py every 30 min.
Script calls staged pipeline from api_proxy.py:
Stage1 base -> Stage2 comment flags -> Stage3 customer -> Stage4 manager -> Stage5 product/price -> Stage6 invoice/payment.
Result: ~300 rows are built locally from 1C.
→

2) GitHub transport

Script writes files:
data/kp_runtime_cache.json
data/kp_runtime_meta.json
data/kp_2026_march_april.json
Then auto-executes git add + commit + push.
GitHub main branch becomes the delivery channel.
→

3) Render runtime (serving)

Render deploys latest commit from GitHub.
On startup API reads snapshot from data/kp_runtime_cache.json.
Dashboard reads rows via /api/kp/all.
Background live refresh from Render to 1C may timeout at Stage1. In that case API keeps last successful snapshot cache.
Operational meaning: production freshness is currently driven by local scheduled refresh on PC, not by guaranteed direct 1C connectivity from Render.
